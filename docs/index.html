Key Features

🧠

Rainbow DQN

Distributional C51, dueling streams, prioritized experience replay, and N-step returns — all combined in a single agent.

👁️

Self-Attention

Multi-head attention over 7 enemy slots lets the agent reason about all threats simultaneously before acting.

🎓

Expert Bootstrapping

A hand-coded Lua expert system provides behavioral-cloning supervision early in training. The DQN gradually takes over.

⚡

Real-Time Training

Thousands of game frames per second flow through a TCP bridge from MAME into the GPU training loop — no batch pre-collection needed.

🔀

Parallel Clients

Up to 36 MAME instances can connect simultaneously, each playing a separate game and generating independent training data.

📊

Live Dashboard

A built-in web dashboard at localhost:8765 displays real-time training metrics, reward curves, and model statistics.

Architecture

The system is split into two sides: a Lua layer inside MAME that reads game memory and sends state to Python, and a Python training server that runs the neural network and pushes actions back. 

  MAME (Lua)                         Python Training Server
  ──────────────────                 ────────────────────────────────────
  state.lua                          socket_server.py
    Read ~80 memory addresses    ──▶   Receive 195-float state vector
    Build 195-float state vec          Queue frame into AsyncReplayBuffer
  logic.lua                                │
    Expert rule-based action               │ batches (PER)
  main.lua                                 ▼
    Serialize + send via TCP         training.py ──▶ train_step()
    Receive 3-byte action reply            │         C51 Bellman update
    Apply to game controls                 │         Gradient descent (GPU)
                                           ▼
                                     aimodel.py
                                       RainbowAgent
                                       ├─ Enemy attention (8 heads)
                                       ├─ Trunk MLP (384 hidden)
                                       ├─ Dueling streams
                                       └─ C51 output (51 atoms × 44 actions)

State Vector (195 floats)

Each game frame is encoded into a fixed-length vector containing: player segment & position, 7 enemy slots (type, depth, segment, velocity, alive flag, fuseball flags), shot positions, spike heights, level geometry, score delta, lives remaining, and more.

Reward Design

Two complementary signals drive learning:

  • Objective reward — score deltas from killing enemies (computed in Lua).
  • Subjective reward — positioning quality: are you aimed at a threat? Are you about to die? (computed by the expert logic module).

Pre-death frames (last 120 before dying) receive boosted replay priority so the agent learns survivability from near-death experiences.

Quick Start

1. Install Python dependencies

# Requires Python 3.10+ and PyTorch
pip install -r requirements.txt

2. Start the training server

# From repo root
cd Scripts
python3 main.py

The server starts on 0.0.0.0:9999 and opens the live dashboard at http://localhost:8765.

3. Launch MAME instances

# Linux / macOS — launch 4 parallel instances
./startmame.sh 4

# Windows
startmame.cmd

MAME must be installed separately, along with the Tempest ROM (tempest.zip). The Lua autoboot script is passed automatically by the launch scripts.

4. Watch training

Open http://localhost:8765 in a browser to see live reward curves, episode counts, exploration rates, and more. The agent starts by imitating the expert system and gradually discovers its own strategies.

Running Tests

# Full test suite (no MAME or GPU required)
pytest tests/

# Quick pipeline smoke test
python3 Scripts/test_pipeline.py

Project Structure

Scripts/

All Python & Lua source. main.py is the Python entry point. main.lua is the MAME entry point.

tests/

pytest test suite. Covers state extraction, enemy masking, pre-death sampling, avoidance logic, and superzap penalties.

models/

Saved checkpoints (tempest_model_latest.pt). Not committed to git — generated at runtime.

Code/Atari/

Original 6502 assembly source for the Tempest ROM. Reference material for understanding game internals.
